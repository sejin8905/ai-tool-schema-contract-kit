# Buyer Checklist

Before using this kit, confirm:

- You can run a TypeScript project locally.
- You understand this is source code, not a hosted SaaS.
- You do not expect a no-code product.
- You do not expect a full JSON Schema validator.
- You do not expect a complete agent security or compliance system.
- You will review and adapt the checks before production use.
- You will not treat this as proof that an AI agent is safe.
- You will test your own tool registry, tool runtime, and approval flow.

Good fit:

- Indie hackers building AI agents.
- Developers maintaining a tool registry.
- Builders who want local checks before tool changes ship.
- Teams that want a small starter before adopting heavier validation or eval tooling.

Poor fit:

- No-code users.
- Buyers expecting a hosted dashboard.
- Teams needing certified safety, security, privacy, legal, or compliance controls.
- Users expecting automatic validation for every provider or framework.
